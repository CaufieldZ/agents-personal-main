# 发票整理 Skill

把零散的发票、支付凭证整理成可报销的结构。

## 约定

### 输入（source_dir）

用户把原始文件丢进一个目录，通常从 Downloads 拖过来。支持的文件：

| 文件类型 | 示例 |
|---------|------|
| 电子发票 PDF | `dzfp_*.pdf`、`发票-个人.pdf`、`Invoice-*.pdf` |
| 支付宝交易明细 PDF | `支付宝交易明细(*).pdf`、`支付明细.pdf` |
| 支付凭证截图 | `.jpg`、`.png`（支付宝/微信支付截图） |
| 已有的明细 xlsx | 跳过，不覆盖 |

同一供应商的发票 + 支付明细 + 截图可能混杂在一起，也可能多供应商混在一个目录。

### 输出（发票/）

```
发票/
├── 明细模板（detail template）.xlsx    ← 唯一模板，不复制到子目录
├── 给费用类型报销细则.pdf              ← 报销政策（已有）
│
├── API服务费/                          ← 费用类型一级目录
│   ├── {供应商}/
│   │   ├── 明细模板（detail template）_{供应商}.xlsx
│   │   ├── {发票}.pdf
│   │   ├── {支付宝交易明细}.pdf
│   │   └── 支付凭证_{供应商}_{序号}.{jpg|png}
│   └── ...
│
├── 差旅/                               ← 按需建，结构同上
└── 其他/                               ← 杂项
```

费用类型目录已有：API服务费、差旅、其他。不够再加。

### 明细模板填写规则

模板位于 `发票/明细模板（detail template）.xlsx`，第二个 sheet「明细模板（请填写）」：

| 列 | 字段 | 填法 |
|----|------|------|
| B | 发生日期 | `YYYY-MM-DD`，取支付/开票日期 |
| C | 费用事项 | `{供应商}平台{服务类型}`，如 `cubence平台大模型API费用` |
| D | 票据张数 | 对应发票 PDF 数量 |
| E | 币种 | `CNY` 或 `GBP` 等 |
| F | 金额 | 单笔金额，多笔多行 |
| G | 事由描述 | 供应商 + 服务说明；外币加注实际 CNY 付款金额 |

填写后删除第一个 sheet（范例），另存为 `明细模板（detail template）_{供应商}.xlsx` 到对应供应商目录。

### 文件命名规范

| 类型 | 命名规则 | 示例 |
|------|---------|------|
| 支付截图 | `支付凭证_{供应商}_{序号}.{ext}` | `支付凭证_yescode_1.jpg` |
| 已合并发票 | 保留 merged 文件名 | `Invoice-MHFFV0K8-merged.pdf` |
| 电子发票 | 保留原始文件名 | `dzfp_261120...pdf` |
| 支付明细 | 保留原始文件名 | `支付宝交易明细(20260201-20260429).pdf` |
| 明细 xlsx | `明细模板（detail template）_{供应商}.xlsx` | 见上 |

## 执行步骤

1. **扫描输入目录**：列出所有文件，按类型分类（发票 PDF / 支付 PDF / 截图 / 其他）
2. **提取数据**：
   - 发票 PDF：用 `pdfplumber` 提取金额、日期、供应商
   - 发票 PDF 是扫描件（pdfplumber 返回空）：用 Read + analyze_image OCR
   - 支付明细 PDF：用 `pdfplumber` 提取交易列表（日期、商户、金额）
   - 支付截图：用 Read + analyze_image 提取交易列表
3. **识别供应商**：从发票抬头 / 支付商户名匹配，同一供应商的文件归一组
4. **匹配发票与支付**：按日期 + 金额关联发票和支付记录
5. **确定费用类型**：根据供应商/服务内容归类（API服务费、差旅、其他）
6. **填写明细模板**：按上面规则填写 xlsx，删除范例 sheet，保存到对应目录
7. **移动文件**：发票 PDF、支付明细、截图（重命名后）移入 `发票/{费用类型}/{供应商}/`
8. **汇总**：输出整理结果统计（供应商数、总金额、各类型金额）

## 报销政策要点（参考）

- 报销币种与发薪币种一致
- 发票抬头个人即可
- 定期报销不跨年
- 差旅需审批
- 交通费需注明事由

## 边界情况

- **多币种**：外币发票金额列写外币，事由描述栏加注「实际支付 ¥XX」
- **截图重叠**：多张截图可能包含重复交易，按（日期时间, 金额）去重
- **无法识别**：放 `其他/unknown/`，不丢弃
- **已有明细 xlsx**：不覆盖，提示用户确认
