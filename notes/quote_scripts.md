# 话术库（闲鱼代订）

风格：**礼貌 + 简洁 + 沉稳**。不口语化、不客服腔。
直接复制改占位符就发。占位符用 `{X}`。

## 0. 通用风格规则

✗ 禁用：
- 句尾语气词「啦」「呗」「咯」「嘛」「哦」
- 「ok」「您」「为您」「方便的话」「核实后」「请问」（连续套话）
- 短促毛躁动词组合：「我去下」「拍我去操作」

✓ 倾向：
- 完整句式，动作前加「我这边」（"我这边下单 / 出票 / 跟进"）
- 「确认」「告知」「需要」这类稳重动词
- 一次只问一件事，不连珠炮
- 报价之前不假设买家要哪档；报价之后不催（一次提醒为限）
- **不收手机号**，要邮箱；护照只在飞机出票前问；铁路不收乘客信息

---

## 1. 招呼 / 破冰

| 场景 | 话术 |
|---|---|
| 买家「在吗」 | 「在的，发下行程：日期 + 人数 + 出发到达」 |
| 买家「有票吗」（无信息） | 「在的，行程发我看下：日期、人数、出发、到达」 |
| 买家发截图无文字 | 「收到截图，确认下日期和人数」 |

---

## 2. 信息收集（按品类 × 维度）

### 2.1 飞机

只问缺的一项，不复述已知。

| 缺什么 | 话术 |
|---|---|
| 日期 | 「几号出发？」 |
| 模糊日期（"五一"/"下周"） | 「具体哪天，{日期 A} 还是 {日期 B}？」 |
| 单程/往返 | 「单程还是往返？往返的话回程是哪天」 |
| 人数 | 「几位？大人小孩分别多少，小孩几岁」 |
| 出发城市多机场 | 「{城市} 起飞用 {A 机场} 还是 {B 机场}？{倾向建议}」 |
| 到达城市多机场 | 同上 |
| 时间偏好 | 「时间有偏好吗？红眼能接受吗」 |
| 舱位 | 「经济舱可以吧？商务/超经差价大，需要的话单独报」 |
| 航司忌讳 | 「有想避开的航司吗？廉航或某联盟之类」 |
| 中转接受度 | 「直飞还是中转都可以？中转便宜 ¥{差价}」 |
| 出票前要邮箱 | 「出票需要个邮箱，航司发行程单用」 |
| 护照（出票前 / 国际线） | 「拍下后发护照首页：中文姓名 / 拼音 / 出生年月 / 护照号 / 有效期」 |
| 护照过期不够 6 个月 | 「你的护照 {日期} 过期，{目的国} 要求至少 6 个月有效期，建议先续签」 |

**多机场城市追问**（直接读 `data/airport_codes.json` 的 note）：
- 「东京 NRT 成田 还是 HND 羽田？羽田离市区近」
- 「上海 PVG 浦东 还是 SHA 虹桥？国际线一般走 PVG」
- 「首尔 ICN 仁川 还是 GMP 金浦？国际线基本都是仁川」
- 「纽约 JFK / LGA / EWR，哪个？国内多用 LGA，国际 JFK，新泽西方向 EWR」

### 2.2 铁路（新干线）

**不收乘客信息**。新干线是不记名票。

| 缺什么 | 话术 |
|---|---|
| 日期 | 「几号出发？」 |
| 人数 | 「几位？小孩 6-11 岁半价、5 岁以下不占座免票」 |
| 时间 | 「想坐几点的车？时间有偏好吗」 |
| 出发站歧义 | 一般默认东京站，不主动问；买家说品川/上野时确认下 |
| 到达站歧义 | 「{站 A} 还是 {站 B}？」（如新大阪/大阪、新神户/三宫、新函馆北斗/函馆市区） |
| 席位 | **不问**。直接报全档让买家选（见 §3.2） |

### 2.3 酒店

| 缺什么 | 话术 |
|---|---|
| 入住/离店 | 「几号入住，几号退？」 |
| 城市/区域 | 「住哪个区？{选项 A} / {选项 B} / {选项 C}」 |
| 距离 | 「离 {地铁站/地标} 多远能接受？」 |
| 人数 | 「几位大人几位小孩？小孩几岁」 |
| 房型 | 「想要单人 / 双床 / 大床 / 三人间？」 |
| 早餐 | 「含早还是不含？含早大概贵 ¥{差价}/人/天」 |
| 房间数（≥3 人） | 「一间还是两间？日本单房多数限 2 人」 |
| 入住人姓名（日本酒店） | 「日本酒店需要全员护照：拼音姓名 + 国籍 + 护照号」 |
| 邮箱（出票前） | 「给个邮箱，酒店发预订确认用」 |

## 3. 报价（按品类 × 分支）

### 3.1 飞机

| 分支 | 话术 |
|---|---|
| 单程直飞经济 | 「查了一下，{出} → {到} {日期}，¥{价}，确认就拍，我这边出票」（quote.py 自动） |
| 往返经济 | 「{出} ↔ {到} {去日}~{回日}，¥{价}，确认就拍，我这边出票」 |
| 中转便宜 ≥ ¥200 | 「直飞 ¥{A}，中转 ¥{B}（{中转城市} 停 {时长}），便宜 ¥{差价}，看选哪个」 |
| 中转便宜 < ¥200 | 「直飞 ¥{A}，中转省 ¥{差价}，建议直飞」 |
| 拆票（联程不联运） | 「联程 ¥{A}，拆票 ¥{B}（行李需重新托运、第一段延误不赔），便宜 ¥{差价}」 |
| 多档舱位 | 「经济 ¥{A}，超经 ¥{B}，商务 ¥{C}，选哪档」 |
| 廉航或需另购行李 | 「裸价 ¥{A}（不含托运），含 23kg ¥{B}，看你需要」 |
| 多人不同票价 | 「成人 ¥{A}×{N}，儿童 ¥{B}×{M}，婴儿 ¥{C}，合计 ¥{合}」 |

### 3.2 铁路

**默认报全档**（quote.py `--rail-route 出-到`）。

| 分支 | 话术 |
|---|---|
| 全档（默认） | 「查了一下，{出} → {到} {日期}{人数}，自由席 ¥{A} / 指定席 ¥{B} / 绿色车厢 ¥{C}{ / Gran Class ¥{D}}，确认哪档我这边下单」 |
| 买家指定单档 | quote.py `--jpy-price N --seat reserved` 走单档话术 |
| 旺季无自由席 | 「这天是 {黄金周/お盆/年末年始}，没有自由席，最便宜的是指定席 ¥{B}」 |
| 希望号无自由席 | 「希望号这班没有自由席，指定席 ¥{B}」 |
| 含换乘 | 「{出} → {换乘站} → {到}，全程约 {时长}，{席位} ¥{价}」 |
| 往返 | 「去程 ¥{A}，回程 ¥{B}，合计 ¥{合}」（新干线无往返折扣） |

### 3.3 酒店

**只报含税价**。

| 分支 | 话术 |
|---|---|
| 默认 | 「{酒店名} {房型} {夜数}晚，¥{价}（含税），不含早」 |
| 含早 | 「含早 ¥{A}，不含早 ¥{B}，差 ¥{差}/晚」 |
| 多间 | 「2 间双床 ¥{A}，1 间四人房 ¥{B}，看你需要」 |
| 不可退 vs 可退 | 「免费取消 ¥{A}，不可退 ¥{B}（便宜 ¥{差}），行程定了再考虑不可退」 |
| 五星 / 民宿要信用卡担保 | **拒单或换酒店**，不向买家要卡号 |

## 4. 比价 / 砍价 / 犹豫

| 场景 | 话术 |
|---|---|
| 「便宜点」 | 「这价含税到手，再低做不了」 |
| 「别家更低」 | 「对方是裸价 / 不含{行李/餐/税}？对比清楚我可以让 ¥{X}」 |
| 让步空间小 | 「最多再让 ¥{X}，到 ¥{新价}，确认就拍」 |
| 「我想想」 | 「这价今天有效，确认了告诉我」 |
| 报价后超 4h 无回 | 「在吗？这价今天有效，看下是否需要」（一次为限） |
| 谈不拢 | 「这价是底了，做不下来，理解，下次有合适的再来」 |

## 5. 拍下后 / 出凭证

| 场景 | 话术 | 触发 |
|---|---|---|
| 拍下确认（飞机） | 「收到，下单中，1-2 小时出 PNR 发你」 | 闲鱼系统提示已下单 |
| 拍下确认（铁路） | 「收到，下单中，30 分钟内发二维码 / 取票码」 | 同上 |
| 拍下确认（酒店） | 「收到，下单中，30 分钟内发确认号」 | 同上 |
| 上游慢 | 「上游还在处理，再等 {X} 分钟，有进展第一时间发你」 | 等待超预期 |
| 出 PNR 回执 | 「PNR：{X}，可以在航司官网办理值机/选座，有问题随时联系」 | send_receipt.py |
| 出铁路票回执 | 「票出好了：{二维码/取票码}，进站直接刷」 | rail 出票后 |
| 出酒店确认 | 「订好了，{酒店} {入退日}，确认号 {X}，前台报姓名或确认号都行」 | hotel 出票后 |

## 6. 出行前 / 行程中

| 场景 | 话术 |
|---|---|
| 飞机出行前 24h | 「明天 {时间} 起飞，建议提前 2 小时到 {机场}{航站楼}，{航司} 柜台办登机」 |
| 铁路出行前 24h | 「{时间} 在 {站} 发车，提前 20 分钟进站」 |
| 酒店出行前 24h | 「{日期} 入住 {酒店}，15:00 入住 / 12:00 退房，前台报姓名或确认号 {X}」 |
| 买家问值机 | 「起飞前 24 小时可以在 {航司} app 选座并办理网上值机，托运行李在柜台办」 |
| 买家问改签 | 「按 {规则}，改签费 ¥{X}，要改的话告诉我新日期，我这边操作」 |
| 行程中航班取消 | 「航司一般会主动改签，到柜台询问，我这边也跟进，有问题随时联系」 |
| 行程中天气延误 | 「天气原因没办法，到机场等通知，飞不了航司会安排后续」 |

## 8. 异常 / 边界

| 场景 | 话术 |
|---|---|
| 报价后未拍涨价 | 「这班价格涨到 ¥{X}，可以改 ¥{X}，或者换 {方案 B} ¥{Y}」 |
| 报价后未拍售罄 | 「这班刚售罄，{方案 B} ¥{X} 还有，要换吗」 |
| 已付款后涨价/缺货 | 「抱歉这班已售罄，可以换 {方案 B} 补差 ¥{X}，或者全额退款」 |
| 姓名拼写不一致 | 「护照显示是 {A}，你发的是 {B}，以哪个为准？航司以护照为准」 |
| 日期搞错（已下单） | 飞机：「改签费 + 差价 ¥{X}」/ 铁路：「新干线纸质票可免费改 1 次」/ 酒店：「免费取消窗口内可重订」 |
| 临时退票 | 「退票费 ¥{X}，能退 ¥{Y}，确认退吗」 |
| 重复下单 | 「你拍重了，我这边退一笔」 |
| 跨日时差陷阱 | 「这班是 {当地日期} 凌晨 {时间}，是不是按 {前一天晚上} 算的？」 |
| 投诉/差评威胁 | 「先看哪里出了问题，能解决就解决；评价是你的权利，能先说下原因吗」 |
| 退款 | 「款项已原路退回」（闲鱼后台「退款」操作完之后） |
| JR Pass 询问 | 「JR Pass 这块我们不做，单纯指定席代订可以」 |
| 信用卡担保酒店 | 「这家需要信用卡预授权，做不了，换 {替代} 怎么样」 |

## 10. 降级流程（截图回复 / 数据不全场景）

用户可能直接发携程/Klook 截图让我看图回复。按四档选最合适的：

| 档 | 触发 | 操作 |
|---|---|---|
| L0 全自动 | 订单数据齐 | `quote.py --order-id X --ctrip-price N --send` |
| L1 模板话术 + 我读价 | 截图给出价格，行程在订单里 | 同 L0，话术走模板 |
| L2 自定义话术 + 落账 | 截图含细节（航司/时刻/转机），需要在话术里体现 | `quote.py --order-id X --price N --quoted-text "..." --send`，台账记 |
| L3 纯回复不落账 | 闲聊、答疑、订单尚未建 | `reply.py --chat-id X --to-uid Y --text "..." --confirm` |

**选档原则**：默认 L2 — 截图情况下细节不能丢，台账也不能漏。只有完全无关订单的对话才走 L3。

**L2 话术写作要点**（参考 §0 风格）：
- 把截图里的航司、班次时刻、转机点写进话术
- 「查了一下，{出} → {到} {日期}，{航司} {时刻}{ 转 {城市}}，¥{价}，确认就拍，我这边出票。」
- 多档同样可手写：「查了一下，{路线}，{日航 ¥A 10:40 / 国航 ¥B 14:25}，看选哪个。」

## 9. 命令对照（话术对应脚本）

| 触点 | 命令 |
|---|---|
| 看新事件 | `python3 commands/inbox.py` |
| 飞机/酒店报价 | `python3 commands/quote.py --order-id X --ctrip-price N` |
| 铁路单档报价 | `python3 commands/quote.py --order-id X --jpy-price N --seat reserved` |
| 铁路全档报价 | `python3 commands/quote.py --order-id X --rail-route 东京-长野` |
| 改价 | `python3 commands/quote.py --order-id X --price N` |
| 一键报价 + 发 | `python3 commands/quote.py --order-id X ... --send` |
| 发 quote.py 生成的话术 | `python3 commands/reply.py --order-id X --use-quoted-text --confirm` |
| 自定义文本回复 | `python3 commands/reply.py --order-id X --text "..." --confirm` |
| 查机场 | `python3 scripts/show_airport.py {码或城市}` |
| 查新干线 | `python3 scripts/show_shinkansen.py 出 到` |
| 回填买家 uid | `python3 scripts/backfill_buyer_uid.py --apply` |
| 发到 TG 发单群 | `python3 commands/dispatch.py --order-id X --confirm` |
| 关联上游 PNR | `python3 commands/match_pnr.py --pnr-file F --order-id X --pnr CODE --confirm` |
| 发 PNR 回执 | `python3 commands/send_receipt.py --order-id X --confirm` |
| 手动推进状态 | `python3 commands/mark.py --order-id X --status S` |
